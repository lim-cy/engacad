Making This Website!

Downloading Github Desktop made updating the website a lot easier. I didn't need to keep uploading files to Github and it only took a few quick clicks to update the files in Github. As it automatically senses any changes I made, I only need to commit and then push to make the change.


I used the Bootstrap CSS framework to have my images and text side-by-side in the projects post(it's used for most of my formatting). Bootstrap is also used for the navigation bar. Since the Projects page is in a blog layout, I intend to just keep adding new "posts" by copy-pasting and then editing sections of the code for new entries. I used Google Fonts for more font options. (Hi, so I'll rewrite the website using Bootstrap 4/5 but for now, it's using Bootstrap 3, sorry!) So I'll document my process rewriting both the HTML and CSS later _(꒪ཀ꒪」∠)_

Now for the in-depth on how I built this. Let's start with the navigation bar! "navbar-fixed-top" makes the navigation bar stay fixed at the top of your screen regardless of scrolling, moving along with it.(Use "navbar-static-top" if you would rather have the navigation bar not move) The three "span class" tags make the three line menu that you will see on mobile view (Try viewing my website on mobile! It looks good passable!) Engineering Academy is under "navbar-brand" so that I can edit it separately from the list items Use "a href=" to link all the text to their respective pages and that's all for the navigation bar!

Now let's talk about the footer! Nothing really special here. Most of my CSS is in my CSS script so you won't see much here. The "script src" are for my Bootstrap CSS framework. Without, it the 3 line menu wouldn't work on mobile but that's about what it does for my website for now.

On to the text that you see in both my Home page and About Page! Pretty much standard stuff. "class=img-responsive" makes it such that the images will scale accordingly to device size. (Again, try it on mobile to see the effect! Or just resize your browser it works too) The "del" tags basically do strikethroughs. I think I am abusing them.

Finally how one of my "posts" look like on the Project Page. Same stuff as before. Lots of divs for formatting, can be a bit hard to keep track but quite easy as long as I comment where it starts and ends. The "div id=white" is to change the background colour of the posts which if you scroll down will see alternate between grey and white. That's it for how I built this site! I don't think I'll be covering the CSS unless it's required?

Fusion360 Work

Here's some of the Fusion360 work that I have done. There's a laser-cut handphone holder that is shaped like a TV set and a 3D-printed desktop organiser. I learnt a lot about Fusion360 through this two assignments.

Commands I learnt:

  • Rectangular Pattern - Super useful for making any type of woodwork joints, essentially a copy paste function in any direction
  • Check for interference(under Inspect) - When trying to piece together parts for a laser cut object, helpful for checking for interference so that you know whether hole/parts are correctly sized/angled
  • Shell - Best tool for making drawers enough said
  • Offset Plane - Useful when used in conjunuction with align for moving objects to a set position that is hard to do by mouse alone
  • Motion Study - Really nice to see how joints move
  • Animation - Animate how your object should be assembled
